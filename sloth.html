SLOTH

It only watches.

A passive terminal network monitor for Linux. Sloth never injects packets, never scans, and never modifies kernel state — it observes what your host already sees and turns it into 27 views, six alert rules, and a WiFi SIGINT toolkit.

▸ View on GitHub
— What it is —

Most network tools are loud. They scan. They probe. They leave fingerprints on every host they touch. Sloth does none of that. It is built on a single discipline: only watch.

That discipline has a payoff. A passive monitor can run on production systems no active tool is welcome on, and the data it captures is the data the host is actually seeing — not a synthesized model of it. Sloth turns that raw observation into twenty-seven ncurses views, six alert rules, a JSONL forensic stream, and an embedded WiFi SIGINT toolkit.

It is written in C99. It builds with or without ncurses. With or without libpcap. On Linux, BSD, and embedded targets. The test suite carries 2,122 assertions and a 51% mutation kill rate.

— Highlights —
  1. Passive by design.

    No injection. No scanning. No kernel modification. Sloth reads what the host already sees, and stops there.

  2. Twenty-seven views, one TUI.

    Interfaces, connections, packets, processes, DNS, TLS with JA3, QUIC, HTTP, ARP, mDNS, NBNS, DHCP, SSDP, ICMP, NTP — tab-cycled in a single ncurses dashboard.

  3. Six alert rules.

    Port scans, deauth floods, NXDOMAIN bursts, threat-intel domain hits, threat-intel IP hits, periodic beaconing. All derived from the same observation pipeline.

  4. WiFi SIGINT toolkit.

    Per-MAC PNL aggregation, RSN/cipher/AKM/MFP inventory from beacons, EAPOL and PMKID capture with hashcat-22000 export, hidden-SSID reveal, and sequence-number MAC-randomization deanonymization.

  5. JSONL stream and SIEM forwarder.

    Read-only Unix or TCP socket for live consumers. Reference Python forwarders ship for Splunk HEC, RFC 5424 syslog, and Elasticsearch Bulk.

  6. Builds anywhere.

    C99, with or without ncurses, with or without pcap. Linux, BSD, and embedded targets. The mutation harness lives in-tree.

— Getting started —
# 1. Build (full: ncurses + pcap + nl80211)
make

# 2. Or build headless / embedded
make embedded

# 3. Run with a JSONL forensic log
sudo ./sloth -o /tmp/sloth.jsonl

# 4. For WiFi SIGINT, set an iface to monitor mode externally —
#    sloth never touches link state itself.
sudo iw dev wlan1 set type monitor
sudo ./sloth --eapol-dir /tmp/sloth-eapol
